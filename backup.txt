INSTRUCCIONES PARA RESTAURAR LA BASE DE DATOS
===========================================

1. PREPARACIÓN
--------------
   - Asegúrate de tener Python y Django instalados
   - Tener el entorno virtual activado
   - Estar en el directorio del proyecto

2. PASOS PARA RESTAURAR
----------------------
   a) Detener el servidor Django si está corriendo
   
   b) Limpiar la base de datos actual (opcional pero recomendado):
      python manage.py flush
      (Responder 'yes' cuando pregunte)
   
   c) Restaurar los datos del backup:
      python manage.py loaddata backup_db.json
   
   d) Verificar la restauración:
      - Iniciar el servidor Django: python manage.py runserver
      - Acceder a la aplicación y verificar que los datos se han restaurado correctamente

3. SOLUCIÓN DE PROBLEMAS
------------------------
   Si encuentras errores durante la restauración:
   
   a) Verificar que el archivo backup_db.json existe y no está corrupto
   
   b) Asegurarte de que todas las migraciones están aplicadas:
      python manage.py migrate
   
   c) Si hay errores de dependencias, intentar:
      python manage.py migrate --run-syncdb
   
   d) Si persisten los problemas, revisar los logs de error

4. NOTAS IMPORTANTES
-------------------
   - Hacer una copia de seguridad antes de restaurar
   - Verificar que la versión de Django es compatible
   - Asegurarte de tener todas las dependencias instaladas
   - Mantener una copia segura del archivo backup_db.json

5. CONTACTO
----------
   En caso de problemas, contactar al administrador del sistema.
